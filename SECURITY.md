# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

If you've found a security vulnerability in Orion-Nginx, we appreciate you taking the time to report it privately. Please follow these guidelines to submit your report.

### How to Report

To report a security vulnerability, please use [GitHub Security Advisories](https://github.com/CTU-SematX/Orion-Nginx/security/advisories/new) or open an issue with the following information:

1. **Description**
   - Provide a detailed description of the security vulnerability.
   - Include as much information as possible to help us understand and address the issue.

2. **Reproduction Steps**
   - Step-by-step instructions to reproduce the vulnerability.

3. **Impact**
   - Describe the potential impact of the vulnerability.

4. **Public Disclosure**
   - Indicate whether this vulnerability has already been publicly discussed or disclosed.
   - If so, provide relevant links.

## Confidentiality

We kindly ask you to keep the report confidential until a public announcement is made.

## Response Timeline

- We will respond within a few weeks to confirm whether your report has been accepted or rejected.
- Vulnerabilities will be handled on a best-effort basis.
- You will be notified when the vulnerability is fixed.

Thank you for helping to improve the security of Orion-Nginx!
